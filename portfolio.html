Sistema ETL de Tráfico Urbano

Una solución End-to-End para la ingesta, procesamiento y análisis de datos de movilidad en tiempo real utilizando arquitectura moderna.

Apache Airflow 2.7 Docker & Compose Terraform (IaC) Python & Pandas SQL & Medallion

Resumen del Proyecto

Este proyecto implementa un pipeline de datos automatizado que transforma datos crudos de sensores de tráfico, semáforos y cámaras en insights accionables. Diseñado bajo la arquitectura Medallion (Bronze, Silver, Golden), garantiza la calidad y trazabilidad del dato. La infraestructura es inmutable y reproducible gracias a Terraform, mientras que Airflow orquesta las tareas complejas de dependencia y reintento.

Infraestructura como Código

Gestión de recursos Docker mediante Terraform para garantizar reproducibilidad.

# main.tf - Definición de recursos
resource "docker_container" "airflow_scheduler" {
  name  = "trafico_airflow_scheduler"
  image = docker_image.airflow_custom.name
  
  env = [
    "AIRFLOW__CORE__EXECUTOR=LocalExecutor"
  ]
  
  volumes {
    host_path      = abspath("${path.module}/../../dags")
    container_path = "/opt/airflow/dags"
  }
}

Valor para el Negocio

  • Toma de decisiones en tiempo real:

    Permite a las autoridades ajustar semáforos basándose en datos actuales, no históricos.

  • Reducción de costos operativos:

    Automatiza la generación de reportes que antes tomaba horas de trabajo manual.

  • Escalabilidad garantizada:

    La arquitectura basada en contenedores permite procesar más sensores sin cambiar el código.
