Apache Airflow Docker Terraform Python

Resumen del Proyecto

Este proyecto implementa una arquitectura de datos completa (End-to-End) para procesar información de tráfico urbano. Utiliza el patrón Medallion (Bronze/Silver/Golden) para transformar datos crudos de sensores en insights de negocio, orquestado automáticamente y desplegado mediante infraestructura como código.

Implementación Técnica

Infraestructura Híbrida

Despliegue flexible utilizando tanto Docker Compose para desarrollo local rápido como Terraform para la gestión profesional de recursos (Redes, Volúmenes, Contenedores).

Archivo: main.tf 
resource "docker_container" "airflow_scheduler" {
  name  = "trafico_airflow_scheduler"
  image = docker_image.airflow_custom.name
  env = [
    "AIRFLOW__CORE__EXECUTOR=LocalExecutor"
  ]
}

Orquestación ETL

DAGs de Airflow diseñados modularmente. Las tareas están desacopladas y utilizan PythonOperators para la lógica de transformación con Pandas y SQL.

Archivo: trafico_diario_urbano.py 
def crear_analisis_golden(**context):
    cursor.execute("""
        CREATE TABLE golden_analisis_trafico AS
        SELECT s.ubicacion, AVG(s.velocidad)
        FROM silver_sensores s
        GROUP BY s.ubicacion
    """)
